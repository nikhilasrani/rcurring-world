# Phase 3 -- UI Design Contract

> Visual and interaction contract for the game systems phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Context:** This is a Phaser 3 pixel art game (480x320 GBA-style resolution). All UI is rendered via Phaser GameObjects (Graphics, Text, Sprite, Container). There is no HTML/CSS UI layer, no shadcn, no Tailwind. All visual specifications below use Phaser rendering primitives and pixel values appropriate to the game's low-resolution viewport.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Phaser 3 GameObjects -- no web UI framework) |
| Preset | not applicable |
| Component library | none (custom Phaser UI components in `src/ui/`) |
| Icon library | none (programmatic pixel art via pngjs/canvas, per project pattern) |
| Font | `monospace` (system monospace, established in Phases 1-2) |

**Source:** Codebase analysis -- DialogBox.ts, ZoneBanner.ts, TitleScene.ts, InteractionPrompt.ts all use `fontFamily: 'monospace'`.

## Spacing Scale

All spacing values use the 16px tile grid as the base unit. The game viewport is 480x320 at integer pixel coordinates. All spacing MUST be multiples of 2 (favoring multiples of 4 at this low resolution where 8-point scale values would be too coarse for some elements).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 2px | Stroke widths, border offsets, fine label offsets only (NOT structural layout spacing) |
| sm | 4px | Dialog box margin, compact padding (established: `BOX_MARGIN = 4`) |
| md | 8px | Default element padding (established: `TEXT_PADDING = 8`) |
| lg | 16px | Tile-sized spacing, tab header height, panel inner padding |
| xl | 24px | Section gaps within pause menu panels |
| 2xl | 32px | Banner height (established: ZoneBanner 32px), major layout spacing |
| 3xl | 48px | Pause menu outer margin from viewport edge |

**Note on `xs` (2px):** This token is an exception to the standard 4px-minimum spacing set. It is strictly reserved for stroke widths (panel borders, selection highlights) and sub-pixel visual offsets (icon-text alignment nudges). It MUST NOT be used for structural layout spacing such as margins, padding, or gaps between components.

Exceptions:
- Touch button sizes are 40px (established: `BTN_SIZE = 40`) -- non-standard but required for 44px-adjacent touch target accessibility.
- Joystick base is 80px (established: `JOYSTICK_BASE_SIZE = 80`).

**Source:** DialogBox.ts (`BOX_MARGIN = 4`, `TEXT_PADDING = 8`, `BOX_HEIGHT = 72`), ZoneBanner.ts (32px banner height), TouchControls.ts (`BTN_SIZE = 40`).

## Typography

All text uses `fontFamily: 'monospace'`. No additional fonts. Weights are controlled via Phaser `fontStyle` property.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Micro | 8px | normal (400) | 1.2 | InteractionPrompt "A" label, save icon label, HUD quest indicator |
| Body | 10px | normal (400) | 1.4 | Dialogue content, inventory flavor text, journal entries, panel body text |
| Label | 12px | bold (700) | 1.2 | NPC name in dialogue, zone banner text, tab headers, panel headings, item names |
| Display | 14px | normal (400) | 1.2 | Title screen menu options ("Continue", "New Game"), notification text |

**Note on size scale (8/10/12/14):** These 2px increments are constrained by the existing codebase (Phases 1-2 established 8px, 10px, 12px, and 14px as the working type sizes) and by the GBA-style 480x320 canvas resolution, where larger jumps would produce visually disproportionate text. At this pixel density, 2px steps provide meaningful visual hierarchy while remaining legible.

**Constraints:**
- No anti-aliased fonts -- `monospace` at these sizes renders crisply on the 480x320 canvas, which is the desired GBA aesthetic.
- All text uses `setOrigin(0, 0)` unless explicitly centered (titles, banners).
- Word wrap width is always computed as `containerWidth - padding * 2`.

**Source:** DialogBox.ts (12px bold name, 10px body, 10px indicator), ZoneBanner.ts (12px bold), InteractionPrompt.ts (8px), TitleScene.ts (14px prompt, 24px title -- but 24px is title-only, not used in Phase 3 UI).

## Color

All colors are hex values used in Phaser `fillStyle()`, `lineStyle()`, or Phaser.Text `color`/`backgroundColor` properties. The game uses a light-on-dark scheme for overlays and a near-white background for dialogue panels.

| Role | Value | Alpha | Usage |
|------|-------|-------|-------|
| Panel Background (dominant) | `#F8F8F8` | 0.95 | Dialogue box, pause menu panel background, metro map background |
| Overlay Dim (secondary) | `#111111` | 0.7 | Full-screen dim behind pause menu, zone banner background, metro map backdrop |
| Panel Border | `#222222` | 1.0 | 2px stroke on dialogue box, pause menu border, metro map border |
| Primary Text | `#111111` | 1.0 | Dialogue body text, panel body text, item descriptions |
| Secondary Text | `#333333` | 1.0 | NPC name, panel headings, interaction prompt background |
| Muted Text | `#666666` | 1.0 | Page indicator, locked/disabled labels, "???" journal placeholders, greyed stations |
| Highlight / Accent | `#E8B830` | 1.0 | Active tab underline, selected inventory slot border, quest HUD indicator, cursor arrow on choices, active station dot on metro map |
| Surface White | `#FFFFFF` | 1.0 | Zone banner text, title screen text, notification text on dark backgrounds |
| Destructive | `#CC4444` | 1.0 | Overwrite save warning text only |
| Success / Quest Complete | `#44AA44` | 1.0 | "Quest Complete!" fanfare text, completion percentage in journal, save success flash |

**Accent `#E8B830` reserved for:**
1. Active/selected tab underline in pause menu (2px bottom line)
2. Selected inventory slot highlight border (2px stroke)
3. Quest HUD indicator text in screen corner
4. Choice cursor indicator (`>`) in dialogue choice mechanic
5. Current station highlight dot on metro map
6. Item pickup sparkle tint

**60/30/10 Application:**
- 60% `#F8F8F8` -- panel backgrounds, content surfaces
- 30% `#111111` at 0.7 alpha -- overlay dim, dark backgrounds behind banners/notifications
- 10% `#E8B830` -- interactive focus indicators, active state markers

**Source:** DialogBox.ts (`#F8F8F8`, `#222222`, `#111111`, `#333333`, `#666666`), ZoneBanner.ts (`0x111111` at 0.7, `#ffffff`), InteractionPrompt.ts (`#ffffff` on `#333333`). Accent `#E8B830` is new -- a warm gold that complements the GBA pixel art palette and provides clear contrast against both light panels and dark overlays.

## Interaction Contracts

### Pause Menu

| Input | Desktop | Touch | Behavior |
|-------|---------|-------|----------|
| Open | Escape key | Tap hamburger button (top-right) | Show dark overlay + tabbed menu; emit MOVEMENT_FREEZE |
| Close | Escape key or B key | Tap X button (top-right of menu) or B button | Hide menu; emit MOVEMENT_FREEZE(false) |
| Switch tab | Left/Right arrow keys | Tap tab header | Active tab underlined in `#E8B830`; panel content swaps |
| Navigate items | Up/Down arrow keys | Tap item directly | Selected item highlighted; details shown below grid |
| Confirm/Select | Enter/Space/A | Tap | Activates focused element (save button, toggle) |

### Dialogue Choice (quest accept/decline)

| Input | Desktop | Touch | Behavior |
|-------|---------|-------|----------|
| Move selection | Up/Down arrow keys | Tap option directly | `>` cursor in `#E8B830` moves between 2 options |
| Confirm choice | Enter/Space | Tap A button or tap option | Selected choice emitted via DIALOGUE_CHOICE event |
| Page does NOT auto-advance | -- | -- | Choice pages block until explicit selection |

### Inventory Grid

| Input | Desktop | Touch | Behavior |
|-------|---------|-------|----------|
| Select slot | Arrow keys (2D grid nav: left/right move within row, up/down between rows) | Tap slot | 2px `#E8B830` border on selected slot |
| View item | Selection auto-shows detail | Tap shows detail | Item name (12px bold) + flavor text (10px) below grid |
| Empty slot | Cannot select | No tap response | Renders as `#CCCCCC` outlined 24x24 square |

### Metro Map

| Input | Desktop | Touch | Behavior |
|-------|---------|-------|----------|
| Open | A button on metro map wall interactable | A button on metro map wall | Metro map overlay appears at depth 70 |
| Close | B key or Escape | Tap X button | Map overlay dismissed |
| Select station | Up/Down arrows | Tap station dot | Locked stations show "Coming in future update" in `#666666` |
| Travel | A on unlocked station | Tap unlocked station | 3-part transition: doors (0.5s), ride (1.5s), doors (0.5s) |

### Title Screen (extended)

| Input | Desktop | Touch | Behavior |
|-------|---------|-------|----------|
| Navigate menu | Up/Down arrows | Tap option | `>` cursor on selected option |
| Select | Enter/Space | Tap | Continue loads save; New Game checks for overwrite |
| Continue (no save) | -- | -- | Option visible but greyed (`#666666`), not selectable |

## Layout Specifications

### Pause Menu Layout

```
+--------------------------------------------------+
|  [X]  QUESTS | INVENTORY | JOURNAL | SAVE | SETTINGS  |  <- Tab bar: 16px tall
|  ___________                                           |     Active tab: #E8B830 underline
```
