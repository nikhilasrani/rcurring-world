# Phase 2 -- UI Design Contract

> Visual and interaction contract for the world interaction phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Note:** This project is a Phaser 3 pixel art game rendered entirely on an HTML5 canvas. There is no HTML/CSS component library, no shadcn, no Tailwind. All UI elements are Phaser Graphics, Text, and Image game objects positioned within the 480x320 game resolution. The design contract below specifies values in Phaser rendering terms, not CSS.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Phaser 3 canvas game -- no CSS framework) |
| Preset | not applicable |
| Component library | Phaser 3.90.0 built-in (Graphics, Text, Tween, Camera) |
| Icon library | none -- icons are programmatic pixel art (pngjs pipeline) |
| Font | `monospace` (system monospace, established in Phase 1) |

**Rationale:** Phase 1 established `monospace` as the sole font across all scenes (TitleScene, BootScene, NameEntryScene). Phase 2 continues this for visual consistency. If canvas text rendering of monospace at small sizes proves blurry at 480x320, a bitmap font atlas can be generated via pngjs as a polish pass -- this is noted in RESEARCH.md Open Question 3 and is not a blocker.

---

## Spacing Scale

Declared values (must be multiples of 4). All values are in game-resolution pixels (the 480x320 coordinate space, not screen pixels):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Dialogue box border inset, icon-to-edge gap, interaction prompt padding |
| sm | 8px | Dialogue text left/right padding, NPC name to text gap, banner text padding |
| md | 16px | Tile-aligned spacing (1 tile = 16px), dialogue box side margins |
| lg | 24px | Dialogue box content area height per text line (line + spacing) |
| xl | 32px | Zone banner total height (background band), interaction prompt float offset |
| 2xl | 48px | Not used in Phase 2 |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- Dialogue box height is 80px (not a multiple of the scale, but locked by CONTEXT.md as "~25% of 320px screen height")
- Touch target for "tap anywhere to advance dialogue" is full screen (480x320) -- no spacing constraint

## Typography

All text renders via `scene.add.text()` with Phaser's canvas text renderer. `pixelArt: true` is set globally in Phaser config.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 10px | normal (400) | 1.4 (14px) | Dialogue content text, sign text (2 lines visible at once within 80px box) |
| Label | 8px | normal (400) | 1.25 (10px) | Interaction prompt "A" icon text, NPC name sublabel |
| Heading | 12px | bold (700) | 1.33 (16px) | NPC name in dialogue box top-left, zone banner display name |
| Display | 14px | normal (400) | 1.3 (18px) | Not used in Phase 2 (reserved for Phase 3 menu headers) |

**Source:** Sizes 10px, 12px, and 8px are derived from Phase 1 established patterns. Phase 1 used 10px (NameEntryScene labels), 12px (subtitle, confirm text, loading text), 14px (press start), 16px (name prompt), 24px (title). Phase 2 dialogue operates at smaller sizes because the dialogue box is 80px tall and must fit NPC name + 2 lines of content + padding + page indicator.

**Font style object (canonical reference for all Phase 2 text):**

```
Dialogue content:  { fontFamily: 'monospace', fontSize: '10px', color: '#111111', wordWrap: { width: 456 } }
NPC name:          { fontFamily: 'monospace', fontSize: '12px', color: '#333333', fontStyle: 'bold' }
Zone banner:       { fontFamily: 'monospace', fontSize: '12px', color: '#ffffff', fontStyle: 'bold' }
Interaction prompt:{ fontFamily: 'monospace', fontSize: '8px',  color: '#ffffff', backgroundColor: '#333333', padding: { x: 2, y: 1 } }
Page indicator:    { fontFamily: 'monospace', fontSize: '10px', color: '#666666' }
```

## Color

All colors are hex values used in Phaser `scene.add.text({ color })` or `graphics.fillStyle(0xHHHHHH, alpha)`.

| Role | Hex | Phaser Hex | Alpha | Usage |
|------|-----|------------|-------|-------|
| Dominant (60%) | #000000 | 0x000000 | 1.0 | Game background (unchanged from Phase 1), camera fade color |
| Secondary (30%) | #F8F8F8 | 0xF8F8F8 | 0.95 | Dialogue box fill background |
| Accent (10%) | #48A868 | 0x48A868 | 1.0 | Reserved for: page indicator arrow when active, interaction prompt highlight pulse (established GBA green from Phase 1) |
| Dark border | #222222 | 0x222222 | 1.0 | Dialogue box border stroke, interaction prompt background |
| Banner background | #111111 | 0x111111 | 0.7 | Zone banner semi-transparent dark band |
| Text primary | #111111 | -- | -- | Dialogue body text color |
| Text secondary | #333333 | -- | -- | NPC name label color, muted UI text |
| Text on dark | #FFFFFF | -- | -- | Zone banner text, interaction prompt text |
| Text muted | #666666 | -- | -- | Page indicator arrow (more pages), inactive elements |

**Accent reserved for:** GBA green (#48A868) is used ONLY for the active page-advance indicator and the interaction prompt pulse animation. It is NOT used for dialogue box borders, NPC names, zone banners, or any other element. This matches Phase 1 where #48A868 is used only for the loading bar and confirm text.

## Component Inventory

Phase 2 introduces 4 new visual components. All render in the UIScene overlay or as WorldScene game objects.

### 1. DialogBox (UIScene)

| Property | Value | Source |
|----------|-------|--------|
| Position | Bottom of screen, y = 240 (GAME_HEIGHT - 80) | CONTEXT.md: "box takes ~25% of screen height" |
| Size | 472px wide x 72px tall (with 4px inset from edges) | 480 - 8px total margin |
| Background | #F8F8F8 at 0.95 alpha | CONTEXT.md: "white/light box" |
| Border | 2px stroke, #222222 | CONTEXT.md: "dark pixel border" |
| NPC name | Top-left corner, 8px from left edge, 8px from top of box | CONTEXT.md: "NPC name displayed in top-left corner" |
| Text area | Starts 24px from box top (below name), 8px left padding | Fits 2 lines at 10px + 14px line-height |
| Page indicator | Down-arrow character, bottom-right corner of box, #666666 | CONTEXT.md: "Down-arrow indicator when more pages follow" |
| Typewriter speed | 30ms per character | CONTEXT.md: "~30ms per character" |
| Advance behavior | First tap = complete text instantly; second tap = next page; last page second tap = close | CONTEXT.md locked decision |
| Movement freeze | Player movement frozen while visible | CONTEXT.md locked decision |
| Sign mode | Same box but NPC name text is empty/hidden | CONTEXT.md: "Signs use same dialogue box but without NPC name label" |

**Dialogue box pixel layout (480x320 coordinate space):**

```
(4, 244) +-----------------------------------------+ (476, 244)
         | [NPC Name]                              |  <- 12px bold, #333333
         |                                         |
         | Dialogue text line 1 goes here and      |  <- 10px normal, #111111
         | wraps to line 2 if the message is long   |
         |                                       V |  <- page indicator
(4, 316) +-----------------------------------------+ (476, 316)
```

### 2. InteractionPrompt (WorldScene, rendered above NPCs/signs)

| Property | Value | Source |
|----------|-------|--------|
| Content | Letter "A" in a small rounded rectangle | CONTEXT.md: "Small 'A' button icon" |
| Font | 8px monospace, #FFFFFF on #333333 background | Matches touch A button visual language |
| Position | Centered above target tile, offset -10px from tile top (floats above head) | Standard Pokemon-style prompt |
| Visibility | Shown when player is on adjacent tile AND facing the target | CONTEXT.md: "adjacent tile only, player must be facing the target" |
| Animation | Gentle 2px vertical bob (tween, y oscillates +/-1px, 800ms loop) | Draws attention without being distracting |
| Depth | 10 (above sprites, below UIScene) | Ensures visibility above NPC sprites |

### 3. ZoneBanner (UIScene)

| Property | Value | Source |
|----------|-------|--------|
| Background | Full-width (480px) x 32px tall band, #111111 at 0.7 alpha | CONTEXT.md: "white text on semi-transparent dark band" |
| Text | Zone display name, 12px bold monospace, #FFFFFF, centered | Legible, authoritative |
| Slide-in | From y=-32 to y=8, 300ms ease Power2 | CONTEXT.md: "slides down from top of screen" |
| Hold | 2000ms (2 seconds) | CONTEXT.md: "stays 2-3 seconds" |
| Slide-out | From y=8 to y=-32, 300ms ease Power2 | CONTEXT.md: "slides back up" |
| Blocking | Non-blocking -- player can move during banner | CONTEXT.md: "Non-blocking -- player can keep moving" |
| Trigger | Zone boundary entry (zones layer tiles) + building entry/exit | CONTEXT.md: "every zone transition triggers a banner" |
| Repeat | Shows once per zone entry -- does not repeat while in zone | CONTEXT.md locked decision |
| Depth | 50 (above all game elements, part of UIScene) | Must be visible above everything |

**Banner feel:** Dark Souls area name reveal -- understated, elegant. Not a popup. No border, no box shadow, no animation bounce. Clean slide in, clean slide out.

### 4. Camera Fade Transition (WorldScene camera)

| Property | Value | Source |
|----------|-------|--------|
| Fade-out | `camera.fade(250, 0, 0, 0)` -- 250ms to black | CONTEXT.md: "fade to black" |
| Fade-in | `camera.fadeIn(250, 0, 0, 0)` -- 250ms from black | CONTEXT.md: "fade back in" |
| Sequence | Freeze movement -> fade out -> load interior -> fade in -> show zone banner -> unfreeze | RESEARCH.md Pattern 4 |
| Color | Black (#000000) | Matches game background color |
| Duration | 500ms total (250 out + 250 in) | Snappy but visible transition |

## Interaction Contracts

### NPC Interaction Flow

```
1. Player walks adjacent to NPC and faces them
2. InteractionPrompt appears (floating "A" above NPC)           <- WorldScene
3. Player presses A / Enter / Space / taps A button
4. NPC patrol stops: gridEngine.stopMovement(npcId)             <- WorldScene
5. NPC turns to face player: gridEngine.turnTowards(...)        <- WorldScene
6. Player movement freezes                                       <- EVENTS.DIALOGUE_OPEN
7. DialogBox appears with NPC name + first page (typewriter)    <- UIScene
8. Player taps -> complete page OR advance to next page         <- UIScene
9. Last page done -> DialogBox hides                            <- UIScene
10. Player movement unfreezes                                    <- EVENTS.DIALOGUE_CLOSE
11. NPC patrol resumes: gridEngine.moveRandomly(...)            <- WorldScene
12. InteractionPrompt hides (player moved or NPC moved)         <- WorldScene
```

### Sign Interaction Flow

```
1. Player walks adjacent to sign and faces it
2. InteractionPrompt appears (floating "A" above sign tile)
3. Player presses action
4. Player movement freezes
5. DialogBox appears WITHOUT NPC name + sign text (typewriter)
6. Player taps through pages
7. DialogBox hides, movement unfreezes
```

### Building Transition Flow

```
1. Player walks onto door tile
2. Movement freezes immediately
3. Camera fades to black (250ms)
4. Interior tilemap loads (replaces outdoor map)
5. Camera bounds update to interior size
6. Player repositions to interior spawn point
7. Camera fades in (250ms)
8. ZoneBanner slides in with building name
9. Movement unfreezes
```

### Building Exit Flow

```
1. Player walks onto exit door tile
2. Same fade transition as entry
3. Outdoor tilemap restores
4. Player repositions to outdoor door location
5. Camera bounds restore to outdoor map
6. Fade in + zone banner for outdoor zone
```

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Interaction prompt | `A` | Single letter, matches GBA A button convention |
| Zone banner format | `{Zone Display Name}` | e.g. "MG Road / CBD", "Cubbon Park", "Metro Station" |
| Page indicator | Down-pointing triangle character | Unicode: rendered as text, not image |
| Dialogue advance hint | (none -- no explicit hint text) | GBA convention: players know to press A to advance |

### NPC Dialogue Copywriting Guidelines

| Guideline | Rule |
|-----------|------|
| Language | English with 2-3 Kannada words per NPC (source: CONTEXT.md) |
| Kannada words to use | "saar", "guru", "namaskara", "banni", "oota aitha?" -- natural code-switching |
| Tone | Friendly, local, like talking to a real person on the street |
| Length | 2-4 pages per NPC, 2 lines per page. Keep it brief. |
| Character voice | Each NPC has a distinct personality (see character list below) |

**NPC Character Voices (copywriting personality guide):**

| NPC | Personality | Sample opener |
|-----|-------------|--------------|
| Chai-walla (Raju) | Proud vendor, warm, slightly pushy about his coffee | "Namaskara! Best filter coffee on MG Road, saar!" |
| Auto driver | Worldly-wise, complains about traffic, knows every shortcut | "Aye guru, you want auto? Traffic is very bad today..." |
| Jogger | Health-conscious, loves Cubbon Park, morning person | "Good morning! Rain trees here are 100 years old, you know?" |
| Shopkeeper | Nostalgic, compares old and new Bengaluru | "This area used to be all gardens... now all malls only." |
| Guard | Formal, proud of Vidhana Soudha, slightly bureaucratic | "Welcome, saar. This is Vidhana Soudha. Very grand building." |

### Sign Copywriting Guidelines

| Sign Type | Copy Style | Example |
|-----------|------------|---------|
| Road name boards | Formal, bilingual feel (English only for v1) | "M.G. Road" |
| Notice boards | Informational, municipal tone | "Cubbon Park -- Established 1870. Please do not litter." |
| Shop fronts | Casual, commercial | "Raju's Filter Coffee -- Since 1985" |
| Landmark plaques | Historical, factual | "Vidhana Soudha -- Seat of the Karnataka State Legislature" |

### Empty and Error States

| State | Context | Copy |
|-------|---------|------|
| No interactable nearby | Player presses A with nothing in front | (nothing happens -- no feedback, standard GBA behavior) |
| Dialogue already active | Player presses A during active dialogue | Advances dialogue (not an error) |
| Interior load failure | Interior tilemap fails to load | Graceful fallback: do not enter, player stays outdoors (BootScene already has graceful asset error handling from Phase 1) |

### Destructive Actions

None in Phase 2. No data is deleted, no progress is reset, no confirmations are needed.

## Interior Visual Contract

4 interior tilemaps using a shared `interior.png` tileset (16x16 tiles, generated via pngjs).

| Interior | Size | Key Visual Elements | Floor Color |
|----------|------|---------------------|-------------|
| Metro station | 15x10 | Ticket counter, platform edge, metro map poster on wall, benches | Gray tile #808080 |
| Coffee shop | 12x10 | Counter, 3 tables with chairs, menu board, steaming cups (decoration) | Warm wood #8B6914 |
| UB City mall | 15x12 | Marble floor pattern, escalator (decorative), shop fronts, plants | Light marble #E8E0D0 |
| Cubbon Park library | 12x10 | Bookshelves (wall tiles), reading tables, portrait frames, old photos | Dark wood #5C3A1E |

Each interior has:
- A `spawn-points` object layer with player entry position (near door)
- An exit door tile that triggers return to outdoor map
- A `zones` object layer with a single zone covering the entire room
- Collision on walls, furniture edges, and counters
