**AudioManager implementation (`src/systems/AudioManager.ts`):**

Import audio config JSON: `import audioConfig from '../data/audio/audio-config.json';`

The class should NOT import Phaser directly. Receive the scene reference in the constructor and use it for `scene.sound.add()` and `scene.tweens.add()`. Type annotations use `any` for Phaser types to avoid import (same isolation pattern as other managers, but AudioManager does need scene ref for sound creation).

Key fields:
- `private scene: any` (Phaser.Scene, typed as any for test isolation)
- `private bgmTracks: Record<string, any>` (Phaser.Sound.BaseSound instances)
- `private sfxSounds: Record<string, any>` (SFX sound instances)
- `private ambientSounds: Record<string, any>` (ambient sound instances)
- `private currentBGMKey: string | null = null`
- `private currentAmbientOverlayKey: string | null = null`
- `private ambientBaseKey: string | null = null`
- `private musicVolume: number = 0.7`
- `private sfxVolume: number = 0.7`
- `private isDucked: boolean = false`
- `private isInInterior: boolean = false`
- `private currentOutdoorZone: string | null = null`
- `private activeCrossfadeTweens: any[] = []`

Constructor:
1. Store scene reference
2. Call `initSounds()` to create all sound instances via `scene.sound.add(key)`
3. Call `wireEvents()` to subscribe to EventsCenter events

`initSounds()`:
- For each entry in audioConfig.bgm: `this.bgmTracks[name] = this.scene.sound.add(entry.key)`
- For each entry in audioConfig.sfx: `this.sfxSounds[name] = this.scene.sound.add(entry.key)`
- For each entry in audioConfig.ambient: `this.ambientSounds[name] = this.scene.sound.add(entry.key)`

`wireEvents()`:
- `eventsCenter.on(EVENTS.BUILDING_ENTER, () => this.onBuildingEnter())`
- `eventsCenter.on(EVENTS.BUILDING_EXIT, () => this.onBuildingExit())`
- `eventsCenter.on(EVENTS.ZONE_ENTER, (zoneName: string) => this.onZoneEnter(zoneName))`
- `eventsCenter.on(EVENTS.DIALOGUE_OPEN, () => this.duck())`
- `eventsCenter.on(EVENTS.DIALOGUE_CLOSE, () => this.unduck())`
- `eventsCenter.on(EVENTS.NPC_INTERACT, () => this.playSFX('npc-chime'))`
- `eventsCenter.on(EVENTS.PAUSE_MENU_OPEN, () => this.playSFX('menu-open'))`
- `eventsCenter.on(EVENTS.PAUSE_MENU_CLOSE, () => this.playSFX('menu-close'))`
- `eventsCenter.on(EVENTS.ITEM_COLLECTED, () => this.playSFX('item-collected'))`
- `eventsCenter.on(EVENTS.QUEST_COMPLETE, () => this.playSFX('quest-complete'))`

`setScene(scene: any)`: Update scene reference after scene restart. Kill any active crossfade tweens from old scene.

`playBGM(trackName: string)`: If same as currentBGMKey, skip. Otherwise crossfade from current to new track.

`crossfadeMusic(toKey: string, duration?: number)`:
- Cancel all active crossfade tweens via `.stop()` and clear the array
- If currently playing BGM, tween its volume to 0 over `duration` (default: `audioConfig.crossfade.musicDurationMs`), onComplete: stop it
- Get new BGM track, set volume to 0, play with loop: true
- Tween new track volume to `this.musicVolume` over `duration`
- Store both tweens in `activeCrossfadeTweens`
- Update `currentBGMKey`

`playSFX(name: string)`: Get sound from sfxSounds[name], call `.play({ volume: this.sfxVolume })`. For 'dialogue-tick', play at sfxVolume * 0.3 (quieter).

`startFootsteps()`: `this.sfxSounds['footstep']?.play({ loop: true, volume: this.sfxVolume * 0.5 })`
`stopFootsteps()`: `this.sfxSounds['footstep']?.stop()`

`setAmbientZone(landmarkId: string)`:
- Look up in audioConfig.zoneAmbientMap: `const ambientKey = audioConfig.zoneAmbientMap[landmarkId]`
- If same as currentAmbientOverlayKey, skip
- If currently playing overlay, tween volume to 0 over ambientDurationMs, onComplete: stop
- If new key exists and differs from base, start new overlay sound at volume 0, tween to 0.6 over ambientDurationMs
- Update currentAmbientOverlayKey

`startOutdoorAmbient()`: Play city-base ambient as base layer at volume 0.4. Store as ambientBaseKey.

`enterInterior(interiorId: string)`:
- Set isInInterior = true
- Stop outdoor ambient base and overlay
- Look up interior ambient from audioConfig.interiorAmbientMap[interiorId]
- Start interior ambient at volume 0.5
- Crossfade BGM to audioConfig.interiorBGM ('interior')

`exitInterior()`:
- Set isInInterior = false
- Stop interior ambient
- Start outdoor ambient base
- Restore outdoor zone overlay if currentOutdoorZone is set
- Crossfade BGM to audioConfig.outdoorBGM ('outdoor')

`onBuildingEnter()`: Called on BUILDING_ENTER event. Calls `playSFX('door-open')`. Marks pending interior transition (actual enterInterior called when interior ID is known -- this happens via a new event or WorldScene calls it directly after setting mode).

`onBuildingExit()`: Calls `playSFX('door-close')`. Calls `exitInterior()`.

`duck()`: If already ducked, return. Set isDucked = true. Tween currently-playing BGM volume to musicVolume * audioConfig.crossfade.duckLevel (0.5) over duckDurationMs (300). Tween active ambient volumes similarly.

`unduck()`: If not ducked, return. Set isDucked = false. Tween BGM back to musicVolume. Tween ambient back to their target volumes.

`setMusicVolume(v: number)`: Store v as musicVolume (clamped 0-1). If BGM is currently playing and not ducked, set its volume immediately.

`setSFXVolume(v: number)`: Store v as sfxVolume (clamped 0-1).

`getSettings()`: Returns `{ musicVolume: this.musicVolume, sfxVolume: this.sfxVolume }`.

`loadSettings(settings: { musicVolume: number; sfxVolume: number })`: Set musicVolume and sfxVolume from provided values.

`startTitleMusic()`: Play title BGM with loop, no ambient.
`stopTitleMusic()`: Stop title BGM.

`onZoneEnter(zoneName: string)`: The ZoneManager emits the zone display name (e.g., "Cubbon Park"), but we need the landmark ID for ambient mapping. AudioManager needs to map display names to IDs. However, looking at ZoneManager.checkZone(), it emits the `foundName` (landmark.name). We need the landmark ID.

IMPORTANT: ZoneManager emits the landmark `name` (display name like "Cubbon Park"), not the `id` (like "cubbon-park"). The AudioManager needs to handle this by either:
- Adding a reverse mapping from display name to ID in audio-config.json
- OR adding a new event ZONE_ENTER_ID that emits the ID

The simpler approach: Add a `zoneNameToId` mapping in audio-config.json:
```json
"zoneNameToId": {
  "Chinnaswamy Stadium": "chinnaswamy-stadium",
  "UB City": "ub-city",
  "Cubbon Park": "cubbon-park",
  "Vidhana Soudha": "vidhana-soudha",
  "MG Road Metro": "mg-road-metro"
}
```

Then `onZoneEnter(zoneName)` looks up `audioConfig.zoneNameToId[zoneName]` to get the landmark ID, then passes it to `setAmbientZone()`.

`destroy()`: Remove all EventsCenter listeners. Stop all sounds. Clear references.
