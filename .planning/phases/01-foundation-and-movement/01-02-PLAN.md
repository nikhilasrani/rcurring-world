  <action>
Create two PNG spritesheets for the player character (male and female variants). Each spritesheet must follow the Grid Engine walkingAnimationMapping convention:

**Spritesheet layout (per gender):**
- Frame size: 16px wide x 24px tall (chibi GBA style -- big head, small body)
- 4 rows (one per direction): Row 0 = Down, Row 1 = Left, Row 2 = Right, Row 3 = Up
- 3 columns per row: walk frame 1 (left foot), stand/idle, walk frame 2 (right foot)
- Total grid: 3 columns x 4 rows = 12 frames per spritesheet
- Total image size: 48px wide x 96px tall

**Character design:**
- Male: Young explorer, dark hair, blue T-shirt, jeans, small backpack. Chibi proportions (head ~40% of body height).
- Female: Young explorer, dark hair (ponytail), green T-shirt, jeans, small backpack. Same chibi proportions.
- GBA 15-bit color palette: limited colors per sprite (max ~15 colors). No anti-aliasing, no gradients. Hard pixel edges.
- Transparent background (PNG alpha channel).

**Grid Engine animation mapping:**
Grid Engine's `walkingAnimationMapping` uses a number index pointing to the "character index" in the spritesheet. For a 3-column layout, setting `walkingAnimationMapping: 0` means:
- Row 0 (frames 0,1,2) = walking down
- Row 1 (frames 3,4,5) = walking left
- Row 2 (frames 6,7,8) = walking right
- Row 3 (frames 9,10,11) = walking up
- Frame 1 (middle column) in each row = the idle/standing pose

These are placeholder sprites that should look like actual GBA pixel art (not developer rectangles), but they do not need to be polished. Functional and recognizable is the goal.

If programmatic generation is not feasible to GBA quality, create simple but clear pixel art: distinct silhouettes, visible direction facing, clear walk cycle motion. Use a tool like canvas-based generation or hand-place pixels in a buffer.
